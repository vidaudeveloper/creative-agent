# Multi-scene grids (handheld product)

Cut on speech pauses. **Default: every scene is talking-handheld** (person + product in frame, lipsync).

## Default rule (required)

| User said | Shot mix |
|-----------|----------|
| (nothing about showcase) | **100%** talking-handheld lipsync — no product-only / macro / B-roll |
| Explicitly wants 产品展示 / 特写 / 开箱 / B-roll | May insert **1** showcase shot (or as many as they asked) |

Do **not** invent showcase beats “for pacing.”

## When to cut (still talking-handheld)

- TikTok/Reels: every ~5–8s of VO
- Feed ads: every ~8–12s of VO
- Always at Hook → Problem → Solution → CTA **copy** beats (visual stays handheld)

## 15s · 2 scenes (default)

```
1. Hook + problem — mid-shot, product in hand, speaking (lipsync)
2. Solution + CTA — mid-shot, product still in hand, speaking (lipsync)
```

## 30s · 3–5 scenes (default)

```
1. Hook — talking handheld (lipsync)
2. Problem — talking handheld (lipsync)
3. Solution — talking handheld + light product gesture (lipsync)
4. Proof — talking handheld (lipsync)  // NOT product macro
5. CTA — look to camera, product still in hand (lipsync)
```

Minimum: **every default shot** shows product clearly (~15–30% of frame) **while** face is visible for lipsync.

## 60s · 5 scenes (default)

```
1. Hook (lipsync handheld)
2. Problem (lipsync handheld)
3. Discovery / product intro (lipsync handheld — product already in hand)
4. Features (lipsync handheld)
5. Proof + CTA (lipsync handheld)
```

## Opt-in showcase (only if user asks)

Example when user says “加一段产品特写”:

```
… talking handheld shots …
N. Product macro / tilt / unbox hands (optional mux VO or silent)
… back to talking handheld CTA …
```

## Transition rules

1. Cut on breath / sentence end (align with per-shot TTS)
2. Slight framing / background change between talking shots (UGC feel) — still same person + product
3. **Do not** insert B-roll between talking segments unless requested
4. Each scene advances Hook→CTA copy; no filler face-only (product must stay visible)
