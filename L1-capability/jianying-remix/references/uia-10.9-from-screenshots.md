# 剪映 10.9.0 导出流程（截图结论）

环境：Windows · 中文专业版 **10.9.0** · 系统缩放 **150%** · 截图草稿 `mens-vest-remix`  
截图目录：用户本机 `Downloads/10.9/`（1–5.png）

> 以下为**界面可见文案/布局**结论，用于写 v10 profile。AutomationId / FullDescription 仍需在 Windows 上用 Inspect 核对。

## 步骤对照

| 步 | 截图 | 界面 | 自动化要点 |
|----|------|------|------------|
| 1 | 1.png | 首页：左侧「首页」，主区「开始创作」，下方「本地草稿(N)」卡片 | 打开草稿：点本地草稿卡片（标题/缩略图），不是「开始创作」 |
| 2 | 2.png | 编辑页：顶栏中央草稿名，**右上青绿色「导出」** | 点顶栏「导出」；注意黄气泡「知道了」可能挡操作，先关掉 |
| 3 | 3.png | 导出窗标题「导出-{草稿名}」：右侧设置 + 底栏青绿色「导出」/「取消」 | 最终导出点**窗底**青绿色「导出」，不是顶栏那个 |
| 4 | 4.png | 「正在导出」+ 进度条百分比 +「取消导出」；中间有 SVIP 续费广告 | 等进度到完成；勿点「立即续费」 |
| 5 | 5.png | 「导出成功…」：底栏「分享链接」/「发布」/「查看草稿」/**关闭** | 结束点「关闭」（不是旧版 ExportSucceedCloseBtn） |

## 与 ≤6 控制器差异（已知）

| 能力 | ≤6（现逻辑） | 10.9（截图） |
|------|--------------|--------------|
| 打开草稿 | `HomePageDraftTitle:{name}` | 首页「本地草稿」卡片；标题可能只在卡片上，需按可见名匹配 |
| 编辑页导出 | `MainWindowTitleBarExportBtn` | 可见文案「导出」青绿色顶栏按钮 |
| 导出窗 | `Name=导出` | 标题更像「导出-{草稿名}」 |
| 分辨率 | `ExportSharpnessInput` + `1080P` | 下拉默认「原始」；说明为草稿尺寸 1080×1920 |
| 帧率 | `30fps` 等 | 下拉显示「30」（可能无 fps 后缀） |
| 导出中 | 旧 sub_status | 文案「正在导出」「任务处理中…」「取消导出」+ `x%` |
| 成功关闭 | `ExportSucceedCloseBtn` | 底栏按钮「关闭」；另有「发布」「查看草稿」 |
| 干扰 | VIP 弹窗 | 新手黄气泡「知道了」；导出中/成功页 SVIP「立即续费」 |

## 缩放

150% 下**优先 UIA 点击**，不要写死像素坐标。若被迫图像点击，必须按 DPI 换算，且仅作 fallback。

## 建议实现

1. `uia_profiles`: `legacy` / `v10_9`  
2. v10 优先：`Name`/`标题` 含「导出」「关闭」「正在导出」「本地草稿」等中文  
3. 分辨率：默认可跳过（用「原始」）；若要强制 1080P，需再抓下拉项文案  
4. 成功判定：出现「导出成功」或「关闭」按钮；然后点「关闭」  
5. 回首页：编辑页关草稿（标题栏关闭）或「查看草稿」后再回首页（待验）

## 仍需 Windows 上补的

- [ ] 实机：`jy-compile export <草稿名> -o ... --profile v10`
- [ ] Inspect：核对 AutomationId（若有则可写回 legacy 更稳）
- [ ] 草稿卡片标题控件是否等于文件夹名
- [ ] 导出路径控件读取是否稳定（失败时会取 Videos 下最新 mp4）
